# AI 短视频生成工作流规范

## 架构总览

```
用户输入主题
    │
    ▼
脚本编写阶段（独立 prompt）
    │  - 读取 scriptwriting-rules.md 作为写作规范
    │  - 输出完整剧本文件（.md）
    │
    ▼
人工审核 / 修改剧本
    │
    ▼
批量素材生成阶段（独立 prompt）
    │  - 读取本规范作为执行规则
    │  - 读取已审核的剧本文件
    │  - 逐轮次（Round）执行素材生成
    │
    ▼
组装与合并
```

## 一、核心概念

### 最小出品单位：轮次（Round）

- **轮次（Round）** 是剧本的最小出品单位，每轮独立成片
- 多轮可合并为一集长视频
- 场景（Scene）是轮次内部的细分，不独立出品
- 生产调度以轮次为单位，每轮完成后可独立输出视频

### 概念层级

```
集（Episode）
  └── 轮次（Round）← 最小出品单位，独立成片
        ├── 场景 1（Scene）← 轮次内部细分
        ├── 场景 2
        └── 场景 3
```

## 二、执行流程

### 第一步：读取剧本

```python
# 读取剧本文件
# 剧本格式见下方"剧本格式规范"
```

**剧本格式规范：**

```markdown
# 第 X 集：[集标题]

## 轮次 1：[轮次标题]
> 情绪基调：xxx | 闭合方式：xxx

### 场景 1：[场景标题]

**镜头 1：镜头类型 · 角色/物体 · 动作**

画面描述文字...

**镜头 2：镜头类型 · 角色/物体 · 动作**

画面描述文字...

### 场景 2：[场景标题]
...

---

## 轮次 2：[轮次标题]
> 情绪基调：xxx | 闭合方式：xxx

### 场景 1：[场景标题]
...
```

### 第二步：逐轮次执行生产

**生产调度以轮次为单位。** 每轮是一个完整的生产批次。

```
轮次 1（可独立出片）
  ├── 场景 1
  │     ├── 镜头 1 → 生成图片 → 图片审核 → 通过/不通过
  │     ├── 镜头 2 → 生成图片 → 图片审核 → 通过/不通过
  │     └── 镜头 3 → 生成图片 → 图片审核 → 通过/不通过
  ├── 场景 2
  │     ├── 镜头 4 → ...
  │     └── 镜头 5 → ...
  └── 轮次 1 素材就绪 → 可独立输出视频

轮次 2（可独立出片）
  ├── 场景 3
  │     └── ...
  └── 轮次 2 素材就绪 → 可独立输出视频

所有轮次就绪 → 合并为完整长视频
```

### 第三步：单镜头执行流程

```
剧本文字（单个镜头描述）
    │
    ▼
根据景别确定 prompt 生成方式：
    ├── 远景/全景 → 使用简化的 prompt 模板（保留画面构成+氛围）
    └── 中景/近景/特写 → 使用完整的 prompt 模板
    │
    ▼
使用图片生成工具生成图片
    │
    ▼
图片审核（见下方审核标准）
    ├── ✅ 通过 → 保存图片，继续下一个镜头
    └── ❌ 不通过 → 重新生成（见重试策略）
    │
    ▼
审核通过的图片 → 使用视频生成工具生成视频片段
    │
    ▼
视频片段审核
    ├── ✅ 通过 → 保存视频片段
    └── ❌ 不通过 → 重新生成
    │
    ▼
所有镜头完成 → 轮次素材就绪
```

## 三、Prompt 生成策略

### 3.1 基于景别的 Prompt 策略

根据剧本中的景别标注，选择不同的 prompt 生成方式：

**远景/全景 — 简化 Prompt**

保留画面构成和氛围，简化角色细节：

```
A cozy living room at night, warm yellow lamplight, a small orange tabby cat sitting on a wooden desk beside a steaming cup of coffee. Soft shadows, cinematic composition.
```

**中景/近景/特写 — 完整 Prompt**

包含完整的角色外貌描述：

```
Medium shot, a small orange tabby cat with white chest fur, round green eyes, sitting upright on a wooden desk, one paw resting on a notebook beside a steaming cup of coffee. The cat is looking directly at the camera with a curious expression. Warm indoor lighting from a desk lamp, cozy room background slightly blurred.
```

### 3.2 镜头类型关键词

在 prompt 开头添加镜头类型关键词：

| 镜头类型 | Prompt 关键词 |
|---------|-------------|
| 远景 | `Wide establishing shot,` |
| 全景 | `Full scene shot,` |
| 中景 | `Medium shot,` |
| 近景 | `Close-up shot,` |
| 特写 | `Extreme close-up,` |

## 四、图片审核标准

### 4.1 空间一致性检查

- [ ] 角色位置关系是否与剧本描述一致？
- [ ] 物体位置是否合理？
- [ ] 空间比例是否正确？

### 4.2 角色一致性检查

- [ ] 角色外貌描述是否符合人设卡？
- [ ] 服装是否正确？
- [ ] 表情是否符合剧本要求？
- [ ] 动作是否与剧本描述一致？

### 4.3 逻辑检查

- [ ] 动作是否有前因后果？
- [ ] 是否存在矛盾（如：左手/右手不一致）？
- [ ] 物体状态是否正确（如：杯子是否在手中）？

### 4.4 画面质量检查

- [ ] 是否有多余的肢体（多指、多手）？
- [ ] 面部是否扭曲？
- [ ] 画面是否模糊或有明显瑕疵？

## 五、重试策略

```
第 1 次失败 → 重新生成
第 2 次失败 → 调整 prompt，增加细节描述
第 3 次失败 → 调整 prompt，简化描述（减少约束）
第 4 次失败 → 标记为"需人工审核"，继续下一个镜头
```

## 六、产出物

### 单轮产出

每轮完成后可独立输出：

```
output/
  ├── round_1/                    ← 轮次 1（可独立播放）
  │     ├── scene_1/
  │     │     ├── shot_001.png
  │     │     ├── shot_001.mp4
  │     │     ├── shot_002.png
  │     │     └── shot_002.mp4
  │     ├── scene_2/
  │     │     ├── shot_003.png
  │     │     └── shot_003.mp4
  │     └── round_1.mp4           ← 轮次 1 独立视频
  ├── round_2/                    ← 轮次 2（可独立播放）
  │     └── ...
  └── episode_001.mp4             ← 合并后的完整长视频
```

### 资源索引文件

每轮生成一个资源索引：

```json
{
  "round": 1,
  "scenes": [
    {
      "scene_id": 1,
      "shots": [
        {
          "shot_id": 1,
          "description": "镜头描述",
          "image_path": "scene_1/shot_001.png",
          "video_path": "scene_1/shot_001.mp4",
          "status": "approved",
          "retries": 0
        }
      ]
    }
  ]
}
```
