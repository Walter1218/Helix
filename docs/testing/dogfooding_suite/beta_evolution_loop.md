# Helix 终极自我演进闭环架构 (Agentic Evals-to-Improvement Loop)

为了让 Helix 突破“人工优化指令”的瓶颈，实现真正的**自编程智能体自我演进**，我们规划了这一套完全脱离人工干预、具备极强工程收敛性的闭环架构。该架构不仅解决了大模型在长期运行中容易产生的“记忆污染”、“死锁发散”和“过拟合”问题，更定义了一条从轻量级 Prompt 优化走向数据飞轮微调的务实演进路线。

## 1. 核心架构设计 (The Evolution Flywheel)

整个闭环系统由 5 大核心组件咬合而成，形成“发现弱点 -> 脏数据过滤 -> 知识萃取 -> 路由验证 -> 巩固进化”的自动化飞轮。

### 组件 A: 动态评测数据源 (Data-Driven Evaluator)
- **机制**：通过 `generate_cases.ts --daily-expand` 每天自动生成 50+ 个对抗性用例（覆盖 React/Bun/Node 等多环境，以及严格类型/依赖丢失等极端条件）。
- **收敛策略 (Adaptive Sampling)**：系统自动统计近期失败率，将生成权重动态倾斜向大模型的薄弱领域，防止陷入“偏科生”困境。

### 组件 B: 可观测与脏数据净化网关 (Observability & Heuristic Filter)
- **机制**：由底层的 `TraceReporter` 和 `SessionRevert` (快照引擎) 提供毫秒级的沙箱保护和执行树追踪。
- **收敛策略 (Sanitization)**：在将失败 Trace 移交给下一环之前，强制通过一道启发式网关。物理剔除所有因网络超时、环境 OOM 或测试框架本身引起的报错，防止大模型产生“因果倒置”的毒药记忆。

### 组件 C: DSPy 离线 Prompt 编译器 (The Optimizer)
- **机制**：在深夜闲时触发，利用本地廉价算力（或闲置 API），阅读通过净化网关的失败 Trace。基于 DSPy 范式，自动变异并“编译”出 5 个不同版本的候选规则。
- **收敛策略 (Held-out Validation)**：新生成的规则不仅要在失败用例上回测，还必须跑通包含 20 个通用基础任务的“回归测试集”。只有泛化能力未受损的规则才允许存入系统。

### 组件 D: 图谱增强经验库 (Graph-Augmented Memory)
- **机制**：为了解决大模型 Context Window 溢出的问题，系统采用“轻量级图谱路由 + 自然语言规则”的融合形态存储经验。
- **收敛策略 (Routing & TTL)**：
  1. **路由剪枝**：当执行新任务时，顺着图谱关系仅提取相关的 Top-K 规则，避免全局 Prompt 臃肿导致中间遗忘。
  2. **知识代谢**：为每条写入图谱的规则加上 TTL (存活时间) 和权重。当框架版本升级或规则长期未命中时，自动降权并代谢剔除。

### 组件 E: 规则合并与状态防呆 (Consolidation & Deadlock Prevention)
- **机制**：防止系统陷入逻辑发散。
- **收敛策略**：
  1. **状态机熔断**：在 `HybridFSM` 中硬编码最大自愈步数和资源配额，触碰红线立刻掐断并回滚。
  2. **图谱自净**：定期在后台让大模型巡检图谱同一节点下的规则，发现矛盾规则立即强制重写合并，消除逻辑死锁。

## 2. 演进路线图 (Strategic Roadmap)

结合 Helix 作为一个基于 TypeScript 的端侧 Loop 引擎的现实约束，我们拒绝一步到位的“重度炼丹”，而是采取分阶段、逐步沉淀数据的落地策略：

### Phase 1: 坚壁清野 (当前 ~ 3 个月)
- **目标**：跑通纯工程化的评测闭环，坚决不让脏数据进入迭代。
- **触发阈值**：0 数据集。
- **动作**：完善多环境动态沙箱 (`setup.ts`)；上线 Heuristic Filter 和沙箱资源 Cgroups 熔断机制；每天跑自动化测试，打下坚实的基建地盘。

### Phase 2: 敏捷进化 (3 ~ 6 个月)
- **目标**：上线轻量级 Prompt 变异，实现经验图谱挂载。
- **触发阈值**：积累 500 ~ 1,000 个高质量的脏环境测试用例 (`cases.json`)。
- **动作**：引入轻量级标签图谱路由；上线基于 DSPy 思想的夜间离线优化器 (`optimize_prompt.ts`)；强制引入回归测试集，防止应试过拟合。

### Phase 3: 数据飞轮与微调降本 (6 个月以后)
- **目标**：将积累的外挂规则内化为模型的“肌肉记忆”。
- **触发阈值**：积累 10,000 ~ 50,000 条包含 `Chosen` 和 `Rejected` 标记的高质量执行轨迹 (Traces)。
- **动作**：利用这批珍贵的私有工程数据集，通过 DPO 算法微调一个专属的端侧小模型（如 8B/14B）。微调完成后，清空知识图谱里的老旧规则，大模型轻装上阵，进入下一轮更高维度的进化循环。
