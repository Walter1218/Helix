# Helix Work — 工作流平台 UI 设计稿

> **文档版本**：v1.2 | 2026-06-22
> **区分声明**：本文档与 `docs/ide-ui-design.md`（VS Code 扩展 IDE 模式 UI 设计）是**完全独立的两个设计体系**。`ide-ui-design.md` 描述的是 VS Code 侧边面板（300-500px 宽）的代码助手 UI；本文档描述的是 Helix Work 独立桌面应用（默认 1200×800px，可最大化）的工作流平台 UI。两者共享 `@mimo-ai/ui` 组件库和 Tailwind CSS 基础，但布局架构、交互范式、组件尺寸完全不同。
> 
> 命名规则：
> - `ide-ui-design.md` = IDE 模式（代码助手，侧边面板）
> - `helix-work-ui-design.md` = Work 模式（工作流平台，独立窗口）

## 一、设计原则

### 1.1 与 IDE 模式的核心差异

| 维度 | IDE 模式 (`ide-ui-design.md`) | Work 模式 (本文档) |
|------|------------------------------|-------------------|
| **容器** | VS Code 侧边面板（300-500px） | 独立 Desktop 窗口（1200×800+） |
| **用户** | 开发者（代码为中心） | 非开发者用户（任务为中心） |
| **布局** | 纵向流式（消息流从上到下） | 横向分栏（导航 + 内容 + 属性面板） |
| **交互** | 聊天对话 + 代码操作 | 拖拽编排 + 状态监控 + 任务管理 |
| **信息密度** | 极高（纵向压缩） | 中等（横向舒展，卡片式） |
| **导航** | 会话标签 + 模式切换 | 侧边栏导航 + 面包屑 + 路由 |
| **主题来源** | VS Code 原生主题变量 | Tailwind 暗色主题（更自由） |
| **核心隐喻** | 聊天助手 | 工作流编排器 + 任务看板 |

### 1.2 Work 模式设计原则

1. **拖拽即编排**：DAG 编辑器的核心交互是拖拽，所有操作必须支持鼠标拖拽完成，键盘快捷键为辅。
2. **状态可视化**：工作流节点、任务状态、执行进度必须一目了然，不需要用户展开或点击才能看到。
3. **零代码友好**：默认面向非开发者，所有配置项必须有中文标签、说明文字和默认值，不暴露技术细节。
4. **开发者可下探**：通过"高级模式"或"导出脚本"提供代码入口，但不作为默认路径。
5. **实时反馈**：工作流运行时，画布上的节点状态实时更新（颜色变化、动画脉冲、进度数字），SSE 事件驱动。
6. **与 IDE 模式零干扰**：Work 模式是独立应用，关闭 Work 窗口不影响 IDE 中的代码助手会话。

## 四、全局组件规范

### 4.1 侧边导航栏 (SideNav)

```
┌────────┐
│  🏠    │  ← 首页 / Dashboard
│  📋    │  ← 工作流列表
│  🔄    │  ← 运行中任务
│  📚    │  ← 知识库
│  📊    │  ← 可观测性
│  📱    │  ← 飞书会话
│  ──    │  ← 分隔线
│  🌙    │  ← 主题切换（暗色唯一，仅作占位）
│  👤    │  ← 用户/设置
└────────┘
```

**规格**：
- 默认宽度：`64px`（图标模式），hover 展开至 `200px`（文字+图标模式）
- 背景：`--hw-bg-nav`
- 图标尺寸：`24px`（Line Awesome 或 Phosphor Icons）
- 图标颜色：默认 `--hw-fg-muted`，激活态 `--hw-brand`
- 激活态：左侧 `3px` 高亮条 `--hw-brand`，背景 `--hw-bg-panel`
- 悬停态：背景 `--hw-bg-panel`，图标 `--hw-fg-secondary`
- 过渡：`width 0.2s ease-out`，`background 0.15s ease`
- 工具提示：收起时 hover 显示 Tooltip（文字 + 快捷键）

**导航项**：

| 图标 | 名称 | 路由 | 快捷键 |
|------|------|------|--------|
| 🏠 | 首页 | `/dashboard` | `Ctrl+1` |
| 📋 | 工作流 | `/workflows` | `Ctrl+2` |
| 🔄 | 运行中 | `/runs` | `Ctrl+3` |
| 📚 | 知识库 | `/knowledge` | `Ctrl+4` |
| 📊 | 可观测性 | `/observability` | `Ctrl+5` |
| 📱 | 飞书会话 | `/feishu` | `Ctrl+6` |
| ⚙️ | 设置 | `/settings` | `Ctrl+Comma` |

### 4.2 顶部工具栏 (TopBar)

**布局**：

```
┌────────────────────────────────────────────────────┐
│  📋 工作流列表          [搜索框]    [➕ 新建] [⚙️] │
└────────────────────────────────────────────────────┘
```

或（Workflow Editor 模式）：

```
┌────────────────────────────────────────────────────┐
│  ⬅ 返回 │ 我的工作流 / 数据分析自动化 │ [💾] [▶️] [⏹] │
└────────────────────────────────────────────────────┘
```

**规格**：
- 高度：`48px`
- 背景：`--hw-bg-panel`
- 边框底部：`1px solid --hw-border`
- 左侧：面包屑导航（返回按钮 + 路径层级）
- 中间：页面标题（字号 `16px`，加粗，颜色 `--hw-fg-primary`）
- 右侧：操作按钮组
  - 保存：`💾` 图标按钮，禁用态灰显，未保存时显示黄色圆点提示
  - 运行：`▶️` 图标按钮，蓝色 `--hw-brand`，hover 时显示下拉菜单（调试运行 / 后台运行）
  - 停止：`⏹` 图标按钮，红色 `--hw-error`，仅运行时显示
  - 导出：`📤` 图标按钮，导出为 JS 脚本或 JSON，下拉菜单增加「另存为模板」选项
- 按钮间距：`8px`
- 按钮尺寸：`32px × 32px`，圆角 `6px`

### 4.3 状态栏 (StatusBar)


**规格**：
- 高度：`28px`
- 背景：`--hw-bg-base`
- 边框顶部：`1px solid --hw-border`
- 左侧：系统状态（绿色圆点 + 文字）

  - 离线：`🔴` 红色 + "离线 — 点击重连"

**折叠按钮**：
- 位置：面板左侧边缘居中
- 尺寸：`24px × 48px`
- 背景：`--hw-bg-panel`
- 边框：`1px solid --hw-border`
- 图标：`◀`（展开）/ `▶`（折叠）
- 动画：折叠/展开 `0.2s ease-out`

### 4.5 通用卡片 (Card)

**规格**：
- 背景：`--hw-bg-panel`
- 边框：`1px solid --hw-border`
- 圆角：`8px`
- 内边距：`16px`（标准）/ `12px`（紧凑）/ `24px`（宽松）
- 阴影：`none`（暗色主题不需要阴影）
- 悬停态：边框变为 `--hw-border-light`，背景 `--hw-bg-card`

- 头部（可选）：标题 + 操作按钮，底部 `1px solid --hw-border` 分隔

### 4.6 按钮 (Button)

**规格**：

| 类型 | 背景 | 前景 | 边框 | 悬停 | 禁用 |

| **Danger** | `--hw-error` | `white` | `none` | `#dc2626` | `opacity 0.5` |
| **Ghost** | `transparent` | `--hw-fg-muted` | `none` | `--hw-bg-hover` | `opacity 0.3` |

- 尺寸：
  - 小：`24px × 24px`（图标按钮）
  - 中：`32px × 32px`（工具栏）
  - 大：`36px 高`（文字按钮，padding `0 16px`）
- 圆角：`6px`（标准）/ `4px`（紧凑）/ `9999px`（pill）

### 4.7 输入框 (Input)

**规格**：
- 高度：`36px`（单行）/ `auto`（多行，最小 `80px`）

### 4.9 模态框 (Modal)

**规格**：
- 遮罩：背景 `rgba(2, 6, 23, 0.8)`，点击遮罩关闭
- 容器：背景 `--hw-bg-panel`，圆角 `12px`，最大宽度 `560px`（标准）/ `800px`（宽）
- 头部：标题（`18px` 加粗）+ 关闭按钮 `×`
- 内容：padding `24px`
- 底部：操作按钮（右对齐，Primary + Secondary）
- 动画：遮罩 `fadeIn 0.2s`，容器 `scale(0.95) → scale(1) + fadeIn 0.2s ease-out`
- 键盘：`Esc` 关闭，`Enter` 确认（当焦点在确认按钮上）

### 4.10 提示消息 (Toast)

**规格**：
- 位置：右上角，距边缘 `24px`
- 尺寸：最大宽度 `400px`，最小宽度 `280px`
- 背景：`--hw-bg-panel`
- 边框：左 `4px` 颜色条（语义色）
- 圆角：`8px`
- 内边距：`12px 16px`
- 内容：图标 + 标题 + 描述 + 关闭按钮
- 自动关闭：成功 `3s`，错误/警告 `不自动关闭`（需手动关闭）

### 5.2 动画定义

```css
/* 脉冲动画 — 运行中节点 */
@keyframes pulse {
  0%, 100% { opacity: 1; box-shadow: 0 0 0 0 rgba(59, 130, 246, 0.4); }
  50% { opacity: 0.8; box-shadow: 0 0 0 8px rgba(59, 130, 246, 0); }
}

/* 抖动 — 错误/失败 */
@keyframes shake {
  0%, 100% { transform: translateX(0); }
  25% { transform: translateX(-4px); }
  75% { transform: translateX(4px); }
}

/* 滑入 — Toast */
@keyframes slideInRight {
  from { transform: translateX(100%); opacity: 0; }
  to { transform: translateX(0); opacity: 1; }
}

/* 淡入 — Modal */
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

/* 缩放 — Modal 容器 */
@keyframes scaleIn {
  from { transform: scale(0.95); opacity: 0; }
  to { transform: scale(1); opacity: 1; }
}

/* 渐变扫过 — 骨架屏 */
@keyframes shimmer {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}
```

### 7.3a 画布核心交互补充

**多选节点**：
- `Shift+Drag`：矩形框选，框内节点全部选中，选框边框 `--hw-brand` 虚线
- `Ctrl+Click`：点击节点添加到当前选择集，再次点击取消选中
- `Ctrl+A`：全选画布内所有节点，再次按取消全选
- 多选后：所有选中节点边框 `--hw-brand`，外发光 `0 0 0 2px rgba(59, 130, 246, 0.2)`
- 多选操作：批量删除（`Delete`）、批量复制（`Ctrl+C`）、批量对齐（右键菜单）

**复制/粘贴/剪切**：
- `Ctrl+C`：复制选中节点（含配置），复制到剪贴板
- `Ctrl+V`：粘贴节点，位置偏移 `(20, 20)` 于原节点，连线不自动复制
- `Ctrl+X`：剪切节点（复制后删除）
- `Ctrl+D`：原地复制并偏移 `(20, 20)`，不经过剪贴板
- 跨工作流：剪贴板内容在切换工作流后保留，支持跨工作流粘贴

**拖拽视觉反馈**：

| 拖拽场景 | 视觉反馈 | 结果 |
|----------|----------|------|
| 从节点面板拖拽到画布 | 半透明节点副本 `opacity 0.6`，`cursor: grabbing` | 释放后在新位置创建节点 |
| 画布内拖拽节点 | 节点 `opacity 0.8`，阴影 `0 0 0 2px --hw-brand` | 释放后更新位置 |
| 拖拽连线（从输出点拖出） | 虚线 `2px dashed --hw-brand`，跟随鼠标 | 悬停到输入点时，输入点放大至 `12px` 高亮 `--hw-brand` |
| 拖拽到有效连接点 | 连接点外发光 `0 0 0 4px rgba(59, 130, 246, 0.3)`，节点边框 `--hw-brand` | 释放后创建连线 |
| 拖拽到无效连接点 | 连接点红色 `0 0 0 4px rgba(239, 68, 68, 0.3)`，光标 `not-allowed` | 释放后无操作，Toast 提示"类型不匹配" |
| 画布平移（Space+拖拽） | 光标变为 `grab`/`grabbing`，画布跟随移动 | 释放后保持新位置 |

**连线编辑**：
- 点击连线：选中态，颜色 `--hw-brand`，粗细 `3px`
- 连线中点悬浮 `🗑️` 删除按钮（圆形 `24px`，背景 `--hw-error`），hover 时显示
- 双击条件标签：进入编辑模式，内联输入框显示

- 类型不匹配：连线变为红色虚线，节点输入点红色高亮，Tooltip 显示"期望 string，实际 number"
- 悬停连线：线条 `2px` → `3px`，颜色从 `--hw-fg-muted` → `--hw-brand`

- 图片/文件：飞书消息中的图片以缩略图形式显示，点击放大查看；文件显示文件名 + 大小 + 下载按钮

**Session 上下文关联**：
- Bot 回复消息底部可展开「Session 上下文」区域
- 展开后显示：关联的 session_id、运行时间、使用的模型、关键工具调用摘要
- 点击 session_id 跳转 Observability Panel 对应记录
- 默认折叠（保持消息流简洁）

### 10.4 输入与回复

**输入区域**：
- 高度：`auto`（最小 `48px`，最大 `120px`）
- 背景：`--hw-bg-panel`，上边框 `1px solid --hw-border`
- 左侧：多行输入框（`Ctrl+Enter` 发送，`Shift+Enter` 换行）
- 右侧：发送按钮 `➤`（Primary 样式，圆形 `36px`）
- placeholder：「输入回复...」
- 发送提示：输入框下方小字 `12px` `--hw-fg-muted`：「回复将发送到飞书 ${chat_name}」
- 发送中：输入框禁用，显示 Loading Spinner，发送成功后消息立即出现在消息流中

**限制**：
- 仅当用户有飞书 Gateway 配置且当前会话在线时显示输入区
- 离线时：输入区替换为提示条「⚠️ 飞书 Gateway 未连接，无法发送消息」
- 权限控制：无回复权限时输入区隐藏，仅显示消息流

### 10.5 筛选与搜索

**全局搜索**（`Ctrl+F` 或 TopBar 搜索框）：
- 搜索范围：当前会话的消息内容、发送者名称、chat_id
- 搜索结果：高亮匹配文字，支持 `↑/↓` 切换上一个/下一个结果
- 搜索计数：显示「3/12」匹配数

**时间筛选**：
- 标题栏右侧增加「📅 时间」按钮，点击弹出日期选择器
- 选择日期后，消息流跳转到该日期最近的消息，日期标签高亮显示

**消息类型筛选**：
- 右键菜单或筛选按钮：按消息类型过滤（全部/文本/图片/文件/系统）
- 默认显示全部

### 10.6 右键菜单与批量操作

**消息右键菜单**：
- 「复制文本」
- 「复制 chat_id」
- 「跳转到关联 Session」（仅 Bot 消息）
- 「查看原始消息 JSON」（Debug 用）
- 「删除此消息」（仅本地删除，不影响飞书服务器）

**会话列表右键菜单**：
- 「复制 chat_id」
- 「跳转关联 Session」
- 「标记所有消息已读」
- 「导出聊天记录」（Markdown/JSON）
- 「删除会话记录」（ConfirmDialog 确认，仅删除本地视图，不删除飞书消息）

### 10.7 可访问性

- 消息流使用 `role="log"` + `aria-live="polite"`，新消息到达时屏幕阅读器播报发送者名称
- 会话列表项使用 `role="listitem"`，选中态 `aria-selected="true"`
- 输入框发送后焦点回到消息流最后一条消息
- 键盘导航：`↑/↓` 在会话列表中切换（需先聚焦到列表），`Enter` 打开选中会话

### 10.8 空状态

**无会话时**：
- 图标：`📱`，标题「暂无飞书会话」
- 描述「飞书 Gateway 连接后，用户与 Bot 的聊天记录会显示在这里」
- 按钮：「⚙️ 检查 Gateway 配置」Secondary（跳转设置）
- 提示：「💡 提示：飞书 Gateway 需要在设置中配置 App ID 和 App Secret」

**选中会话但无消息时**：
- 聊天详情区域显示 EmptyState：图标 `💬`，标题「暂无消息」
- 描述「该会话暂无聊天记录」
- 无操作按钮

**Gateway 未连接时**：
- 整个页面顶部显示警告条：`⚠️ 飞书 Gateway 未连接，聊天记录可能不完整`
- 背景 `--hw-warning` 10% 透明度，文字 `--hw-warning`，高度 `36px`
- 右侧提供「重试连接」按钮

## 十一、响应式设计

### 11.1 窗口尺寸断点

| 断点 | 宽度 | 布局调整 |
|------|------|----------|
| **Desktop** | ≥ 1200px | 三栏布局（侧边导航 + 主内容 + 右侧面板） |
| **Tablet** | 960px - 1199px | 两栏布局（侧边导航收起 + 主内容 + 右侧面板可折叠） |
| **Compact** | 768px - 959px | 两栏布局（侧边导航收起 + 主内容，右侧面板默认折叠） |
| **Minimum** | < 768px | 最小尺寸限制，不允许继续缩小窗口 |

### 11.2 各页面响应式规则

**Dashboard**：
- Desktop：统计卡片两列，工作流列表全宽
- Tablet：统计卡片两列，工作流列表全宽
- Compact：统计卡片单列，工作流列表全宽，筛选栏换行

**Workflow Editor**：
- Desktop：三栏（节点面板 + 画布 + 属性面板）
- Tablet：节点面板可折叠至图标，属性面板默认折叠
- Compact：节点面板折叠，属性面板折叠，通过浮动按钮打开

**Knowledge Base**：
- Desktop：文档列表 + 右侧预览面板
- Tablet：预览面板默认折叠
- Compact：仅文档列表，点击文档打开 Modal 预览

**Observability**：
- Desktop：Trace 树 + 时间线 + 详情面板
- Tablet：时间线隐藏，详情面板折叠
- Compact：仅 Trace 树，详情通过点击节点展开 Modal

**飞书会话**：
- Desktop：左侧会话列表 + 右侧聊天详情
- Tablet：会话列表宽度缩至 `240px`，聊天详情自适应
- Compact：仅显示会话列表，点击会话后全屏进入聊天详情，左上角返回按钮

### 11.3 可访问性规范

**键盘导航**：
- 所有交互元素必须可通过 `Tab` 键聚焦，焦点顺序符合视觉布局
- 焦点样式：`outline: 2px solid --hw-brand`，`outline-offset: 2px`，不可使用默认浏览器虚线框
- 节点面板、DAG 画布、属性面板、筛选栏等区域使用 `role="region"` 和 `aria-label`
- 模态框打开时焦点自动聚焦到第一个输入元素，关闭后焦点回到触发按钮
- 右键菜单打开时，键盘 `↑/↓` 可切换菜单项，`Enter` 确认，`Esc` 关闭

**屏幕阅读器**：
- 节点状态变化时通过 `aria-live="polite"` 区域播报（如「Agent 节点运行完成」）
- 工作流运行进度通过 `aria-valuenow` / `aria-valuemax` 描述
- 确认对话框使用 `aria-describedby` 关联警告描述
- 空状态区域使用 `role="status"`

**减少动画（`prefers-reduced-motion`）**：
- 当用户系统偏好减少动画时，所有 CSS 动画（脉冲、抖动、滑入、缩放）应禁用或替换为即时切换
- 画布节点移动、自动布局过渡从 `0.3s` 变为 `0s`
- Toast 进入动画变为 `opacity 0.2s`（无位移）
- 滚动行为从 `smooth` 变为 `auto`

**对比度与可读性**：
- 所有文字与背景对比度符合 WCAG AA 标准（≥ 4.5:1）
- 错误文字（`--hw-error`）与背景对比度 ≥ 4.5:1
- 禁用态文字（`opacity 0.5`）仍需保证可读性

## 十三、与 IDE 模式的 UI 对比

### 13.1 关键差异汇总

| 维度 | IDE 模式 (`ide-ui-design.md`) | Work 模式 (本文档) |
|------|------------------------------|-------------------|
| **窗口尺寸** | 300-500px 宽（侧边面板） | 1200×800px（独立窗口） |
| **核心交互** | 聊天对话 | 拖拽编排 + 状态监控 |
| **布局方向** | 纵向流式 | 横向分栏 |
| **导航方式** | 会话标签 + 模式切换 | 侧边栏 + 面包屑 + 路由 |
| **信息密度** | 极高（纵向压缩） | 中等（卡片式舒展） |
| **主题系统** | VS Code 原生变量 | Tailwind slate 暗色 |
| **品牌色** | `#007acc` (VS Code 蓝) | `#3b82f6` (Tailwind 蓝) |
| **节点状态** | 无（无节点概念） | 6 种状态色 + 动画 |
| **拖拽交互** | 无 | 核心交互（节点拖拽、画布平移） |
| **实时反馈** | 消息流渐进式 | 画布节点实时状态更新 |
| **目标用户** | 开发者（代码为中心） | 非开发者（任务为中心） |
| **模式切换** | 6 种模式（Ask/Build/Plan/Compose/Loop/Max） | 无模式切换（单一 Work 模式） |
| **子智能体** | 消息流中折叠卡片 | Trace 树中展开/折叠节点 |
| **可观测性** | 消息流中的 Judge/AlignmentGuard 卡片 | 独立 Observability Panel + Trace 树 + 时间线 |
| **飞书会话** | 无（仅 IDE 内聊天） | 独立飞书会话页面，查看多 chat 历史 |
| **知识库** | 无 | 独立 Knowledge Base 页面 |
| **工作流** | 无 | 核心功能，Workflow Editor + Dashboard |
| **长时间任务** | 绑定 IDE 会话生命周期 | 独立任务队列，跨会话持久化 |
| **运行控制** | 无（单次对话） | 运行/暂停/停止 + 进度条 + 实时状态 |
