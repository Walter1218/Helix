#### 1.3.1 连线编辑设计（建议新增 §7.3a）

```
连线操作:
┌────────────────────────────────────────────────────┐
│                                                    │
```

#### 1.3.2 DAG 合法性检查面板（建议新增 §7.7）

```
┌────────────────────────────────────────────────────┐
│  ⚠️ 工作流验证 — 发现 2 个问题                      │  ← 底部面板（可折叠）
│  ──────────────────────────────────────────────────  │
│  ❌ 循环引用：Agent 节点 A → 条件节点 B → Agent 节点 A  │
│     [定位] [自动修复]                                │
│  ⚠️ 孤立节点：结束节点未连接任何输入                 │
│     [定位] [删除]                                   │
│  ──────────────────────────────────────────────────  │
│  [忽略全部]  [自动修复全部]                          │
└────────────────────────────────────────────────────┘
```

**规格**：
- 触发：保存时自动检查，或手动 `Ctrl+Shift+L` 检查
- 显示位置：画布底部，高度 `auto`（最小 `48px`，最大 `200px`），可折叠
- 错误项：红色圆点 `❌`，描述文字，操作按钮
- 警告项：黄色圆点 `⚠️`
- 定位按钮：点击后画布自动平移并高亮对应节点（外发光 `0 0 0 4px --hw-error`）
- 自动修复：循环引用→断开最后一条连线；孤立节点→删除孤立节点

## 二、外化 vs 展开 建议

当前文档大部分内容采用**外化**（直接平铺）形式，这符合"关键信息一眼可见"的原则。但以下部分建议改为**展开/折叠**形式，以优化信息密度和界面整洁度。

### 2.1 应改为「展开/折叠」形式的 6 项

| 项 | 当前形式 | 建议形式 | 理由 |
|----|----------|----------|------|
| **节点配置 — 高级选项** | §7.4 全部平铺 | 默认折叠「重试策略」「超时」分组 | 90% 用户不需要修改默认值，展开后增加认知负担 |
| **日志输出** | 未设计 | 节点详情中可展开「日志」区域 | 日志通常很长，不应常驻占用空间 |
| **错误详情** | §7.5 failed 节点只显示摘要 | 点击 `❌` 后展开错误堆栈和修复建议 | 错误详情只在失败时需要查看 |
| **引用来源** | §8.4 提到可展开，但未明确标注 | 明确标注为「可展开」区域，默认折叠 | 引用来源可能很长，不应常驻 |
| **分块预览** | §8.5 每个分块默认展开 | 默认折叠，点击 `▶` 展开单个分块 | 分块数量可能很多，全部展开占用空间 |
| **运行历史 — 详细步骤** | §6.5 表格只显示基本信息 | 点击行后展开显示每个节点的详细状态 | 运行历史表格应保持简洁 |

### 2.2 应补充「展开/折叠」交互的 4 项

| 项 | 建议形式 | 理由 |
|----|----------|------|
| **属性面板分组** | §7.4 Agent 节点配置分为「基础信息」「Persona」「输入参数」「输出映射」「高级选项」5 个可折叠分组 | 减少滚动，快速定位到需要编辑的字段 |
| **工作流列表项** | 点击项后展开显示最近 3 次运行历史（内联展开，不跳转页面） | 快速查看运行趋势，无需跳转 |
| **文档列表项** | 点击项后内联展开显示分块数量、索引状态、标签 | 快速预览，无需打开右侧面板 |
| **对齐/吸附设置** | 画布右键菜单中「显示/隐藏网格」「对齐到网格」等选项，默认折叠在「视图设置」子菜单中 | 不常用设置不应占用一级菜单 |

### 2.3 应保持「外化」形式的 6 项（当前设计正确）

| 项 | 理由 |
|----|------|
| **节点状态（6 种状态色）** | 运行时节点状态必须一眼可见，折叠会丢失实时感知 |
| **运行进度条（§7.6）** | 工作流运行时的核心信息，必须常驻 |
| **工作流列表项的状态和成功率** | Dashboard 的核心信息，折叠后无法快速评估 |
| **Dashboard 统计卡片** | 首页概览，必须一眼可见 |
| **Trace 树主干结构** | Observability 的核心导航，折叠后无法追踪执行路径 |
| **筛选栏（§6.4 / §8.2 / §9.1）** | 高频操作，必须常驻 |

### 2.4 展开/折叠的交互规格（建议新增到 §5）

```css
/* 展开/折叠动画 */
@keyframes collapse {
  from { max-height: 500px; opacity: 1; }
  to { max-height: 0; opacity: 0; }
}

@keyframes expand {
  from { max-height: 0; opacity: 0; }
  to { max-height: 500px; opacity: 1; }
}
```

**展开/折叠组件规格**：
- 触发：点击分组标题左侧的 `▶`/`▼` 图标，或点击整个标题行
- 图标：`12px`，颜色 `--hw-fg-muted`，展开时旋转 `90°`（`0.2s ease`）
- 动画：`max-height 0.3s ease-out`，`opacity 0.2s ease`
- 标题行：高度 `36px`，背景 `--hw-bg-panel`，padding `0 12px`，hover 背景 `--hw-bg-card`
- 内容区：padding `12px`（展开时），背景 `--hw-bg-base`
- 默认状态：高级选项默认折叠，基础信息默认展开

## 三、其他细节建议

### 3.1 命名一致性

| 位置 | 当前 | 建议 | 理由 |
|------|------|------|------|
| §7.5 表格 | `completed` | `success` | 与 §3.1 CSS 变量 `--hw-node-success` 和 Badge 类型命名保持一致 |
| §7.5 表格 | `warning` | 无此状态 | 节点状态只有 6 种（pending/running/success/failed/paused/skipped），"warning" 是工作流运行状态，不是节点状态 |
| §4.8 Badge | `Running` | `Running` | 正确，但注意与 `run-running` 区分 |
| §3.1 CSS | `--hw-run-failed` | 已正确 | 无问题 |

### 3.2 性能考虑

| 项 | 建议 |
|----|------|
| DAG 画布节点数量 | 建议软限制 100 个节点，超过后显示警告提示性能下降 |
| 日志查看器 | 虚拟滚动（Virtual Scroll），最多渲染 1000 行，超出后分页 |
| 工作流列表 | 虚拟滚动，超过 50 个工作流时启用 |
| 文档列表 | 虚拟滚动，超过 100 个文档时启用 |
| 实时更新 | 使用 `requestAnimationFrame` 批量更新节点状态，避免 60fps 卡顿 |
| 动画 | 减少同时运行的 CSS 动画数量，优先使用 `transform` 和 `opacity`（GPU 加速） |

### 3.3 可访问性（Accessibility）

| 项 | 建议 |
|----|------|
| 键盘导航 | 支持 `Tab` 键在所有交互元素间导航，焦点高亮 `outline: 2px solid --hw-brand` |
| 屏幕阅读器 | 所有图标按钮必须有 `aria-label`，节点状态必须有 `aria-live="polite"` 通知 |
| 颜色对比 | 确保文字与背景对比度 ≥ 4.5:1（暗色主题下 slate-400 在 slate-800 上约 5.8:1，符合） |
| 动画减弱 | 支持 `prefers-reduced-motion`，脉冲动画变为静态颜色变化 |
| 字体大小 | 支持 `Ctrl+Plus/Minus` 调整 UI 缩放（`125%`/`150%`/`75%`） |

## 四、Review 总结

| 类别 | 数量 | 优先级分布 |
|------|------|----------|
| **查漏补缺** | 15 项 | P0: 5 项（右键菜单、拖拽反馈、连线编辑、多选、复制粘贴）/ P1: 7 项 / P2: 3 项 |
| **外化→展开** | 6 项 | 节点高级选项、日志、错误详情、引用来源、分块预览、运行历史步骤 |
| **命名/性能/可访问性** | 3 类 | 建议补充到文档中 |

**推荐行动**：
1. **高优先级**：补充 P0 遗漏项（右键菜单、拖拽反馈、连线编辑、多选、复制粘贴）→ 这是 DAG 编辑器的核心交互，缺失会导致用户无法正常使用
2. **中优先级**：补充 P1 遗漏项（空状态、批量操作、标签系统、日志查看器、实时视图、撤销历史、DAG 检查）→ 影响用户体验但不阻断功能
3. **低优先级**：补充 P2 遗漏项（搜索高亮、性能指标、导出报告、运行对比）→ 可在后续迭代中补充
4. **结构调整**：将 §7.4 的「高级选项」改为默认折叠，将 §8.5 的分块改为默认折叠，明确标注 §8.4 的引用来源为可展开
