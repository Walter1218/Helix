## 重写 README.md 和 README.zh.md

基于与 MiMo-Code 原始仓库（`XiaomiMiMo/MiMo-Code`, HEAD `ab53d061`）的实际代码对比，更新了项目 README 的中英文版本，明确阐述：

1. **Helix 的定位**：基于 MiMo-Code 引擎的增强分支，面向生产环境的自主代码任务执行与自我进化系统。
2. **核心创新点**：
   - 进化飞轮（Evolution Flywheel）—— `script/dogfooding/` 14 个工具，覆盖用例生成、DPO 导出、Judge 验证门、定时 cron 运行。
   - 可观测性层（Observability）—— `TraceReporter` + `HeuristicFilter` + `AlignmentGuard`，MiMo-Code 完全没有此层。
   - 混合记忆检索（FTS5 + Vector RAG）—— MiMo-Code 仅 FTS5，Helix 增加了 `sqlite-vec` + Embedder + 加权融合排序。
   - 安全隔离增强（Shadow Worktree + AST 过滤 + VFS 沙箱 + AlignmentGuard 收件箱纠偏）。
   - 飞书 IM 原生集成（完全自主模式、自适应超时、实时进度推送）。
3. **解决的问题**：Agent 从 Demo 到生产的工程鸿沟（长任务稳定性、脏数据污染、安全隔离、记忆膨胀、过拟合、不可观测性）。
4. **与 MiMo-Code 的关系**：保留全部引擎能力（多 Agent、Effect 服务层、TUI/CLI/API/MCP 多入口），在之上新增 Helix 层。

## 关键对比结论（代码级）

- MiMo-Code 原始仓库：**无** `observability/` 目录、**无** `script/dogfooding/`、**无** `docs/testing/`、**无** Vector 嵌入、**无** 飞书 Gateway。
- Helix 当前仓库：**新增** 上述全部模块，构成完整的自我进化闭环。
